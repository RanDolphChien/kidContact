# kidContact - 學生聯絡單應用

一個雙語言（繁體中文/英文）的學生聯絡單應用，用於台灣兒童英語補習班。

## 📋 專案概述

**kidContact** 是一個單一檔案的純 HTML/CSS/JavaScript 應用，無需任何建構工具或依賴套件。

- **源代碼文件：** `kidContact.html` 包含所有 HTML、CSS 和 JavaScript
- **進入點：** `index.html` 是一個重新導向至 `kidContact.html` 的頁面
- **後端服務：** Supabase REST API（`kiddb` 資料表）

---

## 🗂️ 資料結構

### 應用狀態模型

應用的整個表單由一個狀態物件表示：

```javascript
{
  date: "日期：115 年 5 月 13 日(三)",        // 聯絡單日期
  reminders: "提醒事項第一行\n提醒事項第二行", // 提醒事項（可多行）
  courses: [
    "Book 1 p.1",
    "Book 2 p.2",
    ...
  ],                                          // 課程清單
  students: [
    { name: "學生姓名", comment: "在英文課" },
    ...
  ]                                           // 學生清單及備註
}
```

### 儲存層次（雙層架構）

| 層級 | 位置 | 用途 | 說明 |
|------|------|------|------|
| **本機快取** | localStorage (`kid_contact_state`) | 快速本機儲存 | 應用關閉後保留編輯內容 |
| **雲端同步** | Supabase `kiddb` 表格 | 遠端備份與同步 | 多裝置間的資料同步 |

### Supabase 資料表結構

**表格名稱：** `kiddb`

| 欄位 | 型別 | 說明 |
|------|------|------|
| `id` | text (主鍵) | 固定值：`'main'` |
| `date` | text | 聯絡單日期 |
| `reminders` | text | 提醒事項 |
| `courses` | JSON | 課程陣列 |
| `students` | JSON | 學生陣列 |
| `updated_at` | timestamp | 最後更新時間 |

## 🚀 使用方法

### 啟動應用

無需任何建構步驟，直接在瀏覽器中開啟即可：

1. **開啟檔案：** 在瀏覽器中開啟 `index.html` 或 `kidContact.html`
2. **登入：** 使用 Supabase 認證帳號和密碼
3. **載入資料：** 應用會自動從 Supabase 或本機快取載入資料
4. **編輯內容：** 點擊「編輯」按鈕開啟編輯模式
5. **保存資料：** 點擊「完成編輯」按鈕保存並同步資料
6. **列印：** 點擊「列印」按鈕以 PDF 形式列印聯絡單

### 編輯模式操作

#### 進入編輯模式
- 點擊頁面右上角「編輯」按鈕
- 頁面會添加藍色虛線邊框，標示所有可編輯欄位

#### 編輯內容
可編輯的欄位包括：

| 欄位 | 說明 |
|------|------|
| 日期 | 聯絡單的日期 |
| 課程清單 | 該週上課的英文課程 |
| 提醒事項 | 給家長的提醒訊息 |
| 學生姓名 | 每位學生的名字 |
| 學生備註 | 每位學生的課堂表現或備註 |

#### 管理學生頁面
- **新增頁面：** 點擊「插入頁面」按鈕在目前頁面後新增
- **刪除頁面：** 點擊「刪除頁面」按鈕移除目前學生
- **複製備註：** 點擊「同步備註」（僅第一頁可用）將第一頁的備註複製至所有其他頁面

#### 管理課程
- **新增課程：** 點擊「插入課程」按鈕新增課程項目
- **刪除課程：** 點擊課程項目旁的「刪除」按鈕移除

#### 結束編輯
- 點擊「完成編輯」按鈕
- 應用會自動：
  - 將日期、課程和提醒事項同步至所有學生頁面
  - 將資料保存至本機快取
  - 將資料同步至 Supabase 雲端

### 登出和重新登入

- 點擊登入頁面的「登出」按鈕結束會話
- 下次開啟應用時，使用相同或不同帳號重新登入
- 應用會載入最新的雲端資料或本機快取

## ⏰ 自動登出功能

為了增強安全性，應用在編輯模式中內置了 **10 分鐘無活動自動登出** 功能。

### 運作機制

| 項目 | 說明 |
|------|------|
| **計時器開始** | 使用者點擊「編輯」按鈕進入編輯模式 |
| **計時器重置** | 任何文字輸入或按鈕點擊 |
| **逾時動作** | 10 分鐘無操作後自動：<br/>1. 結束編輯模式<br/>2. 保存資料<br/>3. 登出帳號<br/>4. 返回登入頁面 |

### 支援的互動

- ✅ 文字輸入（桌機鍵盤、行動裝置鍵盤）
- ✅ 按鈕點擊（滑鼠點擊、觸控點擊）
- ✅ 頁面內任何區域的互動

### 設定逾時時間

在程式碼中修改此常數來調整逾時時間：

```javascript
const IDLE_TIMEOUT_MS = 600000; // 預設 10 分鐘（毫秒）
```

## 📖 重要須知

### 1. 雙語言設計

| 部分 | 語言 | 範例 |
|------|------|------|
| **使用者介面** | 繁體中文 | 「編輯」、「完成編輯」、「列印」 |
| **課程內容** | 英文 | "Book 1 p.1", "Book 2 p.2" |
| **預設備註** | 繁體中文 | "在英文課" |

### 2. A4 列印格式

- **頁面尺寸：** 210mm × 290mm（A4）
- **設計特點：** 每位學生為一頁，便於列印和分發
- **列印提示：
  - 使用「列印」按鈕而非瀏覽器列印功能
  - 確保列印機設定為 A4 紙張
  - 使用 PDF 檔案格式保存，確保格式正確

### 3. 資料同步機制

**資料優先權順序（應用啟動時）：**

1. **Supabase 雲端資料** ← 最優先
2. **localStorage 本機快取** ← 次優先
3. **硬編碼預設資料** ← 最後備用

### 4. 離線模式

- 如果無法連線至 Supabase，應用會使用本機快取資料
- 在離線狀態下編輯的內容會保存至本機快取
- 恢復連線時，應用會同步資料至 Supabase

### 5. 瀏覽器相容性

- **建議使用：** 現代瀏覽器（Chrome、Firefox、Safari、Edge 最新版本）
- **必要功能：**
  - `contentEditable` 支援（內聯編輯）
  - localStorage 支援
  - Fetch API 支援

### 6. 安全性注意事項

- ⚠️ **Supabase 認證資訊：** 硬編碼在應用中，僅供內部使用
- ⚠️ **資料隱私：** 確保只在信任的網路環境中使用
- ⚠️ **定期備份：** 定期檢查 Supabase 雲端備份

## 📝 開發者注意事項

### 修改須知

1. **保持單一檔案結構：** 所有代碼應保留在 `kidContact.html` 中
2. **不添加依賴：** 應用應保持零依賴（無 npm、webpack 等）
3. **保護列印樣式：** `@media print` CSS 對應用功能至關重要
4. **驗證修改：** 修改後務必在瀏覽器中測試列印預覽
